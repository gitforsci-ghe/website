# GHE colour identity styling (issue #6)

## Context

Same session. The issue required sourcing GHE colours rather than
guessing hex values and adopting the rstatszh-k012 structure without
its purple/orange palette.

## Prompt (paraphrased)

Add _brand.yml, expand theme.scss (logo sizing, hidden sidebar title,
code colours, three highlight classes), add the content grid and
link-external-newwindow, and re-theme to GHE colours.

## Outcome

- Colours sourced from the Global Health Engineering group's own
  website repo (Global-Health-Engineering/website): blue #0F4C81
  (links, title hover), orange #ED6D37 (code accent), warm gray
  #5c5751 (slide headings), light #F5F5F5 (code and panel
  backgrounds), peach highlight #FEBC94; the yellow highlight #FAE3B4
  was already in this repo. Pink is a computed tint of the GHE orange
  (lighten 30 percent), marked as derived.
- _brand.yml maps the palette to semantic roles with Atkinson
  Hyperlegible and Source Code Pro.
- The missing sidebar logo (images/ghe-square-logo-vector.svg was
  referenced but never committed) is fixed with the GHE GitHub org
  avatar committed as images/ghe-square-logo.png.
- Deck section backgrounds moved from the old purple #4C326A to GHE
  blue #0F4C81; slides.scss headings, links, task callouts and hand
  classes re-themed; content grid and link-external-newwindow added
  to _quarto.yml; footer and sidebar backgrounds moved to #F5F5F5
  (the footer heart emoji was dropped in passing, per the no-emoji
  writing rule).
