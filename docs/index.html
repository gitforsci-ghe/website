Pre-course work: Git for Scientists

Thank you for your interest in this course. Your course instructor Lars Schöbitz is looking forward to meet you.

We will meet on Wednesday, 15 July 2026 - 09:30 to 14:30 (Europe/Zurich) at ETH Zurich, in the CLD Building, Raum A1. Instructions to find us: https://ghe.ethz.ch/about/how-to-find-us.html

Registration for this workshop is closed. You can still share your interest in a future iteration through this form: https://forms.gle/Zf5fPsPJqUxEoLgKA

Background

AI coding agents are rapidly entering scientific work, and using them with confidence starts with the same collaboration skills that make any research project reproducible and reviewable. This workshop gives you those foundations. You will learn to use Git and GitHub for collaboration in scientific work, so that later you can treat an AI agent as a collaborator whose every change is versioned, documented, and open to review. It is the first step in getting ready for AI-assisted scientific work.

Prior knowledge

For this course, we assume that you have some prior knowledge of R and RStudio. You should be able to:

  • install and load R packages
  • read and run short R scripts
  • edit and run R Markdown / Quarto documents
  • navigate the RStudio IDE
  • identify the four panes of the RStudio IDE (Script, Environment, Files, Console)

What do I need to bring to the workshop?

Please ensure you have the following available:

  • a laptop, connected to wifi
  • a web browser (preferably Chrome or Firefox)
  • a pen and paper (we will be doing some drawings)
  • a completed list of pre-work tasks with intermediate due dates

Prior to the workshop, there is some pre-work for you to do. This pre-work will ensure that we can focus our time together on building collaboration skills instead of setting up infrastructure.

What do I need to prepare before the workshop?

Prior to the workshop, complete the following seven steps:

  • Step 1: Download and install R and RStudio
  • Step 2: Get an account for GitHub
  • Step 3: Complete the pre-course survey by end of Thursday, 09 July 2026.
  • Step 4: Create a personal access token for GitHub
  • Step 5: Install Git
  • Step 6: Check your complete setup by Monday, 13 July 2026, in three short parts (unlocked after the pre-course survey is completed):
    • Step 6a: Clone the setup repository
    • Step 6b: Configure Git and push
    • Step 6c: Install packages and report done
  • Step 7: Read “Why Git? Why GitHub?” from “Happy Git and GitHub for the useR”!

Learning objectives

By the end of this workshop, participants will be able to:

  1. Create and clone a repository, and apply the standard Git workflow from the RStudio IDE: pulling, staging, committing, and pushing changes, including reverting a commit.

  2. Work with branches and pull requests: create a development branch, open a pull request, review and comment on a colleague’s pull request, and merge after review.

  3. Publish a rendered document via GitHub Pages, and draw a concept map of how Git and GitHub fit together as a foundation for collaboration.
